# Skala Front

이 저장소는 HTML, CSS, Vanilla JavaScript만으로 프론트엔드 화면을 구현하면서 UI 구조와 스타일링을 공부한 프로젝트입니다.

> [!NOTE]
> 본 프로젝트는 퍼블리싱 구조와 CSS 설계 방식을 익히고 Js를 통해 Data를 관리하는데 집중했습니다

## 프로젝트 소개

과제에서 요구하는 수준의 페이지 구성에서 프로젝트 설계 관점에서의 고민을 더하여 레이아웃을 어떻게 잡아야 하는지, 어떤 기준으로 spacing을 정해야 하는지, OOCSS 관점에서 어떤 식으로 구조를 나눠야 하는지를 추가로 학습하면서 프로젝트를 진행했습니다.

CSS를 설계하는 관점을 공부하면서 OOCSS에 대해 학습했습니다. 특히 `Container`와 `Context`를 분리해서 생각하고, CSS를 `layout`, `skin`, `interaction`, `animation`으로 나누는 방식으로 작업을 정리해보는 것이 프로젝트 진행에 많은 도움이 되었습니다. 

디자인의 경우, (주)당근의 채용페이지의 디자인 스타일과 컴포넌트에서 영감을 얻었습니다. 디자인 토큰, 디자인 시스템이라는 개념에 대해 학습해나가면서 보다 체계적으로 디자인 자원을 관리해나가야겠다는 배움이 있었습니다.

## 학습 목표

- Semaintic HTML을 적용하여 Layout을 설계하는 방버
- OOCSS 원리를 적용해 재사용 가능한 스타일을 만드는 방법 공부
    - `Container`와 `Context`를 나눠서 UI를 설계하는 연습
    - `layout`, `skin`, `interaction`, `animation`으로 CSS 책임을 분리해보는 실습
    - `semantic HTML`, `body`, wrapper, `container` 단위에서 `margin`과 `padding`을 어떤 기준으로 정해야 하는지 공부

## 공부한 내용

- OOCSS 개념을 바탕으로 객체 중심으로 스타일을 정리하는 방법
- 레이아웃 잡기: 그리드, 간격, 정렬, 구조 설계
- 스킨 잡기: 색상, 폰트, 배경, 테두리, 톤 정리
- 인터랙션 잡기: hover, focus, active 같은 상태 변화 반영
- 애니메이션 잡기: transition, transform, 자연스러운 움직임 표현
- semantic HTML 구조를 고려해서 전체 페이지를 설계하는 방법
- wrapper와 `container` 단위의 spacing 규칙을 정리해 페이지 전체 흐름을 일관되게 만드는 방법


## 프로젝트 구조

```text
.
├── html/      # 페이지 마크업 파일
├── css/       # 공통 스타일과 전체 스타일 시트
├── script/    # Vanilla JavaScript 동작 로직
└── media/     # 이미지, 오디오, 비디오 파일
```

## AI 활용 방식에 대한 사항

이 프로젝트는 부분적으로 AI의 도움을 받으면서 진행했지만, 먼저 직접 시도해보고 문제가 생기면 그 원인을 생각해본 뒤 문제를 반드시 정의하여 답변을 얻는 방식으로 활용했습니다. 

즉, AI가 만들어준 답을 그대로 붙이는 대신, 내가 먼저 고민하고 문제를 정리한 뒤 그 흐름에 맞춰 최종 구조를 잡는 방식으로 진행했습니다.


## 향후 발전 방향

이 페이지는 단순히 화면을 구현한 결과물로 끝내기보다, 앞으로 보완점을 계속 채워가며 포트폴리오 페이지로 활용할 예정입니다. 지금은 HTML, CSS, Vanilla JavaScript 중심으로 구조와 레이아웃을 정리하는 단계지만, 이후에는 서버 통신, 데이터 처리, 인증 흐름, API 연동, 상태 관리 같은 영역까지 확장하면서 Full-stack 엔지니어링 역량을 꾸준히 쌓아갈 계획입니다.

무엇보다 포트폴리오 페이지로 활용할 계획이기에 계속 업데이트하면서 점점 더 넓은 기능과 깊은 구현 역량을 담아가는 방향으로 발전시켜 나갈 생각입니다.
